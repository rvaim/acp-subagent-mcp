# 配置说明

配置文件默认名为 `agents.toml`。也可以通过环境变量指定：

```bash
SUBAGENT_MCP_CONFIG=/absolute/path/to/agents.toml node dist/index.js
```

## `[defaults]`

```toml
[defaults]
timeout_secs = 600
inactivity_timeout_secs = 120
heartbeat_timeout_secs = 60
max_depth = 2
max_prompt_chars = 120000
max_inline_file_chars = 30000
max_tool_output_chars = 12000
default_detail_level = "summary"
log_dir = ".subagent-runs"
session_ttl_secs = 1800
completed_session_ttl_secs = 600
max_active_sessions = 8
```

字段说明：

| 字段 | 含义 |
|---|---|
| `timeout_secs` | 整个任务最大运行时间 |
| `inactivity_timeout_secs` | 多久没有有效进展就判定超时 |
| `heartbeat_timeout_secs` | 多久没有任何存活信号就判定心跳超时，默认 60 秒 |
| `max_depth` | 防止子代理递归调用子代理 |
| `max_prompt_chars` | 渲染后 prompt 最大字符数 |
| `max_inline_file_chars` | 单个 inline/snippet 文件内容最大字符数 |
| `max_tool_output_chars` | MCP tool 返回给主代理的最大文本字符数 |
| `default_detail_level` | 默认输出详情级别 |
| `log_dir` | 日志根目录 |
| `session_ttl_secs` | 活跃 session 最大保留时间，当前主要作为预留参数 |
| `completed_session_ttl_secs` | 已完成 session 最大保留时间，当前主要作为预留参数 |
| `max_active_sessions` | 最大活跃 session 数，当前主要作为预留参数 |

## `[security]`

```toml
[security]
allowed_cwd_roots = ["/Users/me/projects", "/tmp"]
allow_network = false
```

`allowed_cwd_roots` 是最重要的安全配置。所有工具输入中的 `cwd` 都必须在这些根目录下。

建议：

- 开发环境可以写项目根目录。
- 不要直接写 `/`。
- 不要把包含敏感数据的家目录整体放进去。

## `[concurrency]`

```toml
[concurrency]
max_parallel_tasks = 4
default_conflict_policy = "single_writer_per_cwd"
```

冲突策略：

| 策略 | 说明 |
|---|---|
| `allow_readonly_parallel` | 允许只读任务并行；写任务同 cwd 会受限 |
| `single_writer_per_cwd` | 同一个 cwd 同时只允许一个写任务 |
| `sandbox_worktree` | 预留配置，当前未实现真实 git worktree |

## `[session_pool]`

```toml
[session_pool]
enabled = true
max_pooled_sessions_per_parent = 4
max_context_usage_ratio = 0.9
idle_ttl_secs = 1800
reuse_policy = "auto"
```

会话池的作用是复用 ACP session，减少重复初始化成本。

复用必须同时满足：

- `parent_agent_id` 相同。
- `agent_type` 相同。
- `cwd` 相同。
- `mode` 相同。
- `mcp_servers` 签名相同。
- 权限策略签名相同。
- session 空闲。
- 估算上下文使用率低于 `max_context_usage_ratio`。

## `[permissions.default]`

```toml
[permissions.default]
read = "allow"
search = "allow"
edit = "deny"
execute = "deny"
network = "deny"
```

可选值：

| 值 | 含义 |
|---|---|
| `allow` | 自动允许 |
| `deny` | 自动拒绝 |
| `cancel` | 取消任务 |

可以按 agent 覆盖：

```toml
[permissions.codex]
read = "allow"
search = "allow"
edit = "allow"
execute = "deny"
network = "deny"
```

## `[agents.<name>]`

```toml
[agents.fake]
description = "本地示例 ACP agent"
command = "node"
args = ["examples/fake-acp-agent.mjs"]
capabilities = ["analysis", "review", "demo"]
system_prompt = """
你是一个示例子代理。
"""
```

字段说明：

| 字段 | 含义 |
|---|---|
| `description` | 给主代理看的说明 |
| `command` | 子代理启动命令 |
| `args` | 子代理启动参数 |
| `capabilities` | 能力标签 |
| `system_prompt` | 渲染任务 prompt 时的前置说明 |
| `env` | 子代理专属环境变量，会在日志中脱敏 |

子代理必须支持 ACP stdio。也就是说，它要能从 stdin 读取 JSON-RPC 消息，并向 stdout 写回 JSON-RPC 消息。
