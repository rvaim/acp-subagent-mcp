# 目录结构

本文说明每个目录和文件负责什么。读代码时可以把它当地图。

## 顶层目录

```text
@rvaim/acp-subagent-mcp/
  src/                  TypeScript 源码
  doc/                  中文详细文档
  examples/             示例配置和 fake ACP agent
  dist/                 编译后的 JavaScript
  README.md             项目入口说明
  DESIGN.md             架构设计说明
  package.json          npm 脚本和依赖
  tsconfig.json         TypeScript 编译配置
```

## `src/index.ts`

程序入口。

职责：

- 读取 `SUBAGENT_MCP_CONFIG` 环境变量。
- 加载配置文件。
- 创建并启动 MCP Server。

## `src/server.ts`

MCP Server 注册位置。

职责：

- 创建 `McpServer`。
- 注册所有 MCP tools。
- 创建 stdio transport。
- 调用 `connect` 开始服务。

新增工具时通常要改这个文件。

## `src/config/`

配置加载模块。

```text
src/config/loadConfig.ts
src/config/types.ts
```

职责：

- 读取 `agents.toml`。
- 合并默认值。
- 校验配置结构。
- 输出统一的 `ServerConfig`。

## `src/tools/`

MCP tool 输入 schema。

```text
src/tools/schemas.ts
```

职责：

- 用 Zod 定义每个工具的输入格式。
- 限制工具调用者不能传危险字段。

## `src/task/`

任务契约和 prompt/结果处理。

```text
src/task/types.ts
src/task/renderSubagentPrompt.ts
src/task/parseSubagentResult.ts
```

职责：

- 定义 `SubagentTask`、`TaskFile`、`SubagentRunOutput` 等类型。
- 把结构化任务渲染成中文 prompt。
- 把子代理返回文本解析成结构化结果。

## `src/acp/`

ACP 通信模块。

```text
src/acp/AcpClient.ts
src/acp/JsonRpcStdioTransport.ts
src/acp/eventAggregator.ts
src/acp/types.ts
```

职责：

- 启动 ACP agent 子进程。
- 通过 stdio 收发 JSON-RPC。
- 发送 `initialize`、`session/new`、`session/prompt`。
- 收集 `session/update`。
- 处理 ACP 反向请求，例如权限和文件读写。

## `src/runtime/`

运行时编排模块，也是项目最核心的部分。

```text
src/runtime/subagentRuntime.ts
src/runtime/taskRegistry.ts
src/runtime/sessionPool.ts
src/runtime/concurrency.ts
src/runtime/heartbeat.ts
src/runtime/processManager.ts
src/runtime/security.ts
src/runtime/permissions.ts
src/runtime/logs.ts
src/runtime/outputCompaction.ts
src/runtime/redact.ts
```

### `subagentRuntime.ts`

总调度器。

职责：

- 实现所有 MCP tools 的业务逻辑。
- 校验任务。
- 创建日志。
- 启动或复用 ACP session。
- 执行 prompt turn。
- 处理超时、取消、清理和输出压缩。

### `taskRegistry.ts`

任务注册表。

职责：

- 保存正在运行或已经完成的任务。
- 根据 `task_id` 查询任务。
- 通知 `subagent_wait` 有任务状态变化。

### `sessionPool.ts`

会话池。

职责：

- 保存可复用 ACP session。
- 按 `parent_agent_id` 隔离。
- 判断 agent、cwd、mode、权限和 MCP servers 是否兼容。
- 超过上下文使用阈值时不再入池。

### `concurrency.ts`

并发和文件写冲突控制。

职责：

- 限制最大并行任务数。
- 防止多个写任务同时改同一个 cwd。

### `heartbeat.ts`

超时监控。

职责：

- 检查 heartbeat timeout。
- 检查 inactivity timeout。

### `processManager.ts`

子进程管理。

职责：

- 启动 ACP agent 子进程。
- 取消时清理子进程或进程组。

### `security.ts`

安全校验。

职责：

- 校验 `agent_type`。
- 校验 `cwd`。
- 校验文件路径。
- 校验 prompt 和 inline 文件大小。
- 处理递归深度限制。

### `permissions.ts`

权限决策。

职责：

- 读取配置中的权限策略。
- 对 ACP permission request 返回 allow、deny 或 cancel。

### `logs.ts`

日志文件读写。

职责：

- 创建运行日志目录。
- 写 task、prompt、events、stderr、result。
- 读取日志尾部。

### `outputCompaction.ts`

输出压缩。

职责：

- 根据 `detail_level` 裁剪 MCP tool output。
- 避免默认返回完整原始事件。

### `redact.ts`

脱敏。

职责：

- 将 token、secret、password 等字段替换成 `[REDACTED]`。

## `examples/`

示例目录。

```text
examples/agents.toml
examples/fake-acp-agent.mjs
examples/mcp-config.json
```

### `fake-acp-agent.mjs`

本地测试用 ACP agent。

它的作用是验证协议链路，不调用任何真实模型。

### `agents.toml`

本地可运行配置，默认使用 fake agent。

### `mcp-config.json`

给 MCP Host 参考的启动配置。
